<objective>
Build the `ChannelResolver` (plugin `AttributeResolverService`) that resolves `resource.channel.*` attributes — including membership as `resource.channel.members` — and the ABAC seed policies that gate channel read/post/moderation/creation. This plan is where the **D-03 reconciliation (Landmine 1)** lands.

## D-03 RECONCILIATION — READ THIS (Landmine 1)

CONTEXT decision **D-03** literally specifies a `ChannelAttributeProvider` resolving `principal.channel_memberships` (principal-side). **That shape is architecturally infeasible via a plugin** and is superseded by the resource-side model below. Grounding (verified this phase):

- The plugin `AttributeResolverService` proto exposes ONLY `GetSchema` + `ResolveResource` — there is **no** subject/principal resolution RPC (`api/proto/holomush/plugin/v1/attribute.proto:44,63`). `PluginAttributeProvider.ResolveSubject` returns nil (`internal/plugin/attribute_proxy.go:44-47`). A plugin therefore **cannot** contribute `principal.channel_memberships`.
- A core in-process provider cannot read the plugin's isolated `plugin_core_channels` schema either.

**DIRECTIVE (this is the built shape):** Model membership **RESOURCE-side** as `resource.channel.members`, resolved by the plugin's own `ChannelResolver.ResolveResource` — exactly like `resource.scene.participants` (`plugins/core-scenes/resolver.go:74-128`). Shape policies as `principal.id in resource.channel.members`. Read CONTEXT's "`ChannelAttributeProvider`" as "the plugin `ChannelResolver`." This satisfies CHAN-02 (membership-gated read/post) and CHAN-04 (private/admin vs public) with ZERO new proto, and **preserves the D-03 seam intent**: the future per-character faction gate lands as a `principal.faction` CHARACTER attribute (added to `internal/access/policy/attribute/character.go`, NOT a channel-plugin concern), and the private/admin policies are keyed on `resource.name`/`type` so that clause slots in with no schema migration and no policy rewrite.

Purpose: default-deny ABAC is the core security control of this phase.
Output: `resolver.go` + manifest policies/actions/verbs; resolver + policy-eval tests green.
</objective>
