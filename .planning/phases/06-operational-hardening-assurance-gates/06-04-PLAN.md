QUAL-01 (D-07/D-08): reconcile the coverage policy so the documented bar and the
enforced bar agree, and add a project-coverage ratchet.

Framing (do NOT reframe as "add a patch gate" — it already exists): codecov/patch
@ 80% is ALREADY a hard merge gate via the protect-main RULESET (not classic
branch protection — `gh api .../branches/main/protection` returning 404 is

codecov never measures, and (b) there is no project floor so coverage can silently
erode. Fix the docs to the enforced reality and add a no-drop project ratchet that
nudges the ~54.6% baseline upward without retroactively blocking legacy code.

Output: `.codecov.yml` project ratchet, the duplicate `codecov.yml` deleted, the
doc MUST rows rewritten, and a flagged manual ruleset step for the project status.
