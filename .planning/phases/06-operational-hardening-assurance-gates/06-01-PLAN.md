<objective>
OPS-02 atomic core (D-01): convert `events_audit` from a single un-partitioned
table into a RANGE-partitioned table and — in the SAME plan/PR — change the shared
`writeAuditRow` INSERT so the audit write path still dedups correctly under the new
composite primary key.

REDESIGNED after cross-AI review (06-REVIEWS.md consensus blockers 1–3). Three
load-bearing corrections replace the prior DEFAULT-partition / timestamp-flip design:
- **No DEFAULT partition.** The prior design attached the old table as a permanent
  DEFAULT partition — non-viable: Postgres rejects the ATTACH when legacy rows fall in
  the current-month ranges, AND forbids `DETACH PARTITION CONCURRENTLY` while a DEFAULT
  exists (which would make 06-02's retention DDL invalid). The partitioned parent here
  has NO DEFAULT; legacy rows are re-homed by 06-02's one-time backfill.
- **A separate deterministic partition key, not the `timestamp` column.** The key is a
  NEW BIGINT column `event_ms`, computed by a shared helper as
  `ulid.Time(id.Time()).UnixNano()` (NOT `ulid.Time(id)`, which does not compile — see the
  ULID note below), derived purely from the immutable event ULID, so it is IDENTICAL on the
  live path and the DLQ-replay path (no cross-partition duplicate). The existing `timestamp`
  column is left UNTOUCHED (still JetStream store-time) so cold-history window filtering
  keeps its exact current semantics — no tier-boundary drift, no parity test needed.
- **Data-preserving down.** The down migration copies partitioned rows back into the
  restored un-partitioned table BEFORE dropping the partitioned parent, so a rollback
  loses no audit records.

Because `writeAuditRow` is shared by the live projection (`projection.go`) AND DLQ
replay (`replay.go`), the migration PK change, the conflict-target change, and the new
event_ms column are ONE indivisible unit. This plan also trips the mandatory
crypto-reviewer surface (projection.go + events_audit migration), gated by a blocking
pre-push checkpoint (Task 4).

ROUND-2 REVIEW INCORPORATION (06-REVIEWS.md consensus, verified against live source):
- **ULID helper compile fix (finding 1):** `event_ms` is computed via a SHARED package
  helper `eventMsFromULID(id) = ulid.Time(id.Time()).UnixNano()`. `oklog/ulid/v2@v2.1.1`
  defines `func Time(ms uint64) time.Time` and `func (id ULID) Time() uint64`, and
  `decodeULIDString` (projection.go:511) returns `([]byte, error)` — so the round-1 form
  `ulid.Time(parsedID)` does NOT compile. Task 2 defines the helper; 06-02's Backfill reuses it.
- **Legacy PK/index rename (finding 2):** migration 000009 named `events_audit_pkey` and
  the `events_audit_*` indexes; `RENAME TABLE` does not free those names. Task 1 renames the
  legacy PK + all `events_audit_*` indexes to `_legacy` BEFORE creating the new parent, and
  the acceptance asserts index/constraint OWNERSHIP (which table), not mere name existence.
- **Test/seed blast radius (finding 3):** `event_ms NOT NULL` + composite `ON CONFLICT`
  breaks the ~16 test/seed files that INSERT directly into events_audit (production is fine —
  only writeAuditRow inserts). Task 3 sweeps them so `task test:int` stays green.
- **Co-deploy with 06-02 (finding 4):** 000052 renames all history off `events_audit`; it
  MUST ship in the SAME PR as 06-02's Backfill (a history outage otherwise).
- **06-PATTERNS.md was REFRESHED (finding 12)** to this revised design (separate event_ms
  key, no DEFAULT, Go backfill, timestamp untouched) — executors read it via context.

ROUND-4 REVIEW INCORPORATION (06-REVIEWS.md Consensus Summary, operator decision made):
- **Live-deployment migration window (round-4 HIGH — operator chose OPTION A, deploy CHOREOGRAPHY,
  NOT expand/contract):** the production deploy runs `core migrate` BEFORE recreating services, so
  after 000052 the still-running OLD core's `INSERT ... ON CONFLICT (id)` with no `event_ms`
  (projection.go:414) would fail on BOTH the NOT NULL event_ms and the dropped id-alone unique index.
  The operator's decision is to KEEP this single-step migration UNCHANGED and make the deploy safe by
  choreography: 06-02 Task 5 re-orders `.github/workflows/deploy.yaml` + the manual runbook to stop the
  old core before migrate, start the new core, and gate restoring traffic on the new core's readiness
  (its synchronous audit backfill gate). So THIS plan does NOT change 000052 — it records the accepted
  bounded audit-write window as a deliberate ops risk (must_haves truth + threat T-06-1-08) and points
  at 06-02 Task 5 for the choreography. Do NOT redesign 000052 as expand/contract.

Output: migration `000052_*` (partition-swap, idempotent, data-preserving down), the
`writeAuditRow` idempotency fix + shared ULID helper, the test/seed sweep, proven by an
integration test that the same event through live + replay produces exactly one row under
the composite PK.
