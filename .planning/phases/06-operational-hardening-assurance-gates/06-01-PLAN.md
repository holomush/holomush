<objective>
OPS-02 atomic core (D-01): convert `events_audit` from a single un-partitioned
table into a RANGE-partitioned table (partition key `timestamp`), and — in the
SAME plan/PR — change the shared `writeAuditRow` INSERT so the audit write path
still dedups correctly under the new composite primary key.

Purpose: this is the single highest-risk edit in the phase. Partitioning forces
`PRIMARY KEY (id, timestamp)`, which breaks the current `ON CONFLICT (id) DO
NOTHING` dedup. Because `writeAuditRow` is shared by the live projection
(`projection.go`) AND DLQ replay (`replay.go`), a partial change breaks every
audit write on deploy. The migration PK change, the conflict-target change, and
the timestamp-source change are ONE indivisible unit.

Output: migration `000052_*` (partition-swap, idempotent, paired down) plus the
`writeAuditRow` idempotency fix, proven by an integration test that the same
event through live + replay produces exactly one row.
</objective>
