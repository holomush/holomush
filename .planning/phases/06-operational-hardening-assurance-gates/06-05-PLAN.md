<objective>
OPS-04 (D-05/D-06): fix the audit-DLQ replay CLI for external-NATS deployments
and replace its tautological coverage test with a genuine recovery assertion.

The F3 bug: the CLI reads `event_bus.game_id` from operator config; when that is
empty, `dlqConfigForGame("")` yields an empty-Subject config that `Defaults()`
fills to `internal.main.audit.dlq`. But the server persists its game_id (usually
a ULID) and publishes to `internal.<ULID>.audit.dlq`, so the CLI's prefix never
matches â†’ every dead letter is counted Failed and nothing is recovered.

Fix: resolve the effective game_id (--game-id override â†’ persisted DB value â†’
config) before building the DLQ config, mirroring the server's own resolution
order. Replace the tautological test (same `"main"` on both seed and replay,
embedded NATS) with a natstest-backed test that exercises the divergent-game
mismatch path and asserts both the failure guard and recovery.

Depends on 06-01 so the rewritten integration test runs against the final
composite-PK / deterministic-timestamp writeAuditRow regime (the DLQ replay path
shares writeAuditRow; its dedup now keys on (id, timestamp)).
</objective>
