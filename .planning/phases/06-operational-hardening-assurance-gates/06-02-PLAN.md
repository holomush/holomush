<objective>
OPS-02 worker (D-01/D-02): make audit retention actually happen. Add a genuine
`events_audit` PartitionManager implementation, wire the (currently unwired)
`RetentionWorker` into the production lifecycle so it prunes on a cycle, and add
the operator-configurable retention window (default 90d).

Purpose: the migration in 06-01 made `events_audit` partitioned, but nothing
DETACHes/DROPs old partitions and nothing maintains the forward window. The
existing `RetentionWorker` (`internal/audit/retention.go`) has ZERO non-test
callers and the full `PartitionManager` interface has NO production impl (only
`EnsurePartitions` exists on the ABAC `access_audit_log` creator; the rest are
test mocks). This plan writes the real impl and starts the worker.

Output: `retention_partitions.go` (PartitionManager for events_audit), the
worker started inside `SubsystemAuditProjection`, and a `RetainWindow` config
knob threaded from operator config into `audit.Config.Defaults()`.

ROUND-2 REVIEW INCORPORATION (06-REVIEWS.md consensus, verified against live source):
- **Shared ULID helper (finding 1):** Backfill computes event_ms via 06-01's shared
  `eventMsFromULID(id) = ulid.Time(id.Time()).UnixNano()` (same package) — NOT the
  round-1 `ulid.Time(id)` form (a compile error: Time takes uint64 ms).
- **Start ordering / dual-copy window (finding 10):** the synchronous boot gate runs
  BEFORE the projection starts — construct manager → Backfill → EnsurePartitions → then
  newProjection/start → then the periodic worker. Today Start starts the projection first
  (subsystem.go:229-236), which lets a DLQ replay race a not-yet-backfilled legacy row.
- **First-cycle scope (finding 9):** the boot gate is Backfill + EnsurePartitions ONLY —
  NOT a full RunOnce (which also Detach/Drops via retention.go:63-98); pruning is left to
  the periodic worker so a first boot on a red deploy never prunes.
- **Detach→rename crash atomicity (finding 7):** DETACH CONCURRENTLY runs outside a tx and
  the rename is a separate statement; each detach cycle first RECONCILES canonical-named
  events_audit_YYYY_MM tables that are no longer children (crash-orphaned) into the
  _detached_<unix> form so they cannot be permanently stranded.
- **Bootstrap orphan-check gap (finding 5):** runBootstrapOrphanCheck (core.go:296, BEFORE
  audit Start) scanned only events_audit; after 000052's rename, residual rows sit in
  events_audit_unpartitioned. Task 4 extends the check to also scan that table when present.
- **Co-deploy with 06-01 (finding 4):** ships in the same PR (this Backfill restores
  cold-history visibility). **06-PATTERNS.md was refreshed (finding 12).**

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
