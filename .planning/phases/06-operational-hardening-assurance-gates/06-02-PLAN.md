## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| retention worker → events_audit partitions (DDL) | The worker issues DETACH/DROP; a wrong-window or wrong-table drop destroys audit records |
| operator config → retention window | An operator-supplied window governs how long audit history is retained (compliance-relevant) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-2-01 | Denial/Tampering | DetachExpiredPartitions/DropDetachedPartitions | high | mitigate | Detach only children whose UPPER event_ms bound <= olderThan; never touch events_audit_unpartitioned or non-child tables; drop only detached children whose `_detached_<unix>` name-age exceeds grace; integration test proves recent partitions survive and within-grace detached ones are not dropped |
| T-06-2-02 | Tampering | manager targets wrong table | medium | mitigate | DDL literals hardcode `events_audit`; compile-time interface satisfaction; test seeds/asserts against events_audit specifically |
| T-06-2-03 | Denial | first-cycle DDL failure leaves projection half-up | medium | mitigate | Run backfill + first EnsurePartitions SYNCHRONOUSLY in Start so a first-cycle failure returns an error and rolls back the projection (the async worker Start could not surface this); bounded drain in Stop; drain test green |
| T-06-2-04 | Denial | replayed historical event has no covering partition (unprunable sink) | medium | mitigate | EnsurePartitions covers the retention window backward + forward; there is no DEFAULT sink (06-01), so an out-of-window replay fails loud (message retained) rather than piling into an unprunable partition |
| T-06-2-05 | Repudiation | over-aggressive window silently discards required audit history | low | accept | Window is operator-configured with a conservative 90d default; documented as adjustable; not auto-shrinking |
</threat_model>
