<objective>
Create `internal/focuscontract`, a neutral types-only leaf package holding the focus contract
that `internal/plugin` needs, and convert the original declarations in `internal/grpc/focus`
into type aliases pointing at it.

This plan is **purely additive plus aliasing** — it changes no behavior, moves no logic, and
touches no consumer. `internal/plugin` is deliberately NOT rewired here (that is 08-02), so
this plan cannot break the plugin tree.

Purpose: D-09 seam 1. `internal/plugin` currently imports *up* into `internal/grpc/focus` on
7 production edges; ARCH-02 cannot produce a clean layering while the manager imports its own
consumer's subpackage. This plan builds the destination; 08-02 redirects the arrows.

Output: a new leaf package + alias re-exports, with the whole repo still compiling unchanged.
</objective>

**RESEARCH.md § Seam 1 lists 5 declarations. The verified transitive type closure is 7.**
`RestorePlan.Streams` is `[]StreamWithMode` (`internal/grpc/focus/kind_policy.go:30`), and
`StreamWithMode.Mode` is `ReplayMode` (`internal/grpc/focus/kind_policy.go:19`, itself an alias
to `session.ReplayMode`). A 5-symbol move does not compile. Both extra symbols are in this
plan's scope. Verified live this session with `sed -n '105,140p' internal/grpc/focus/coordinator.go`
and `rg -n 'type ReplayMode|type StreamWithMode' internal/grpc/focus/`.

**Design decision — alias, don't move-and-rewrite.** `internal/grpc/focus` has many in-package
users (`defaultCoordinator`, `KindPolicy`, `join.go`, `restore.go`, `subscription_router.go`, …)
plus consumers in `internal/grpc` and `internal/web`. Declaring the originals as Go type aliases
(`type Coordinator = focuscontract.Coordinator`) makes the relocation transparent: alias types are
*identical*, not merely convertible, so every existing reference site, every method set, and every
compile-time assertion continues to hold with zero edits. This is what keeps the seam a
zero-churn, zero-behavior-change cut.
</planner_corrections>
