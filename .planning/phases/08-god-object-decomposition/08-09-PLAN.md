**The invariant binding covers the import-direction half ONLY.** `INV-PLUGIN-56` is allocated (next
free in the `INV-PLUGIN` scope; 55 is the current maximum, verified live). It names a durable
architectural guarantee — the same shape as `INV-WORLD-4`, which binds
`TestWorldPostgresCompositionAllowlist`. The size-ceiling half stays **unbound**: it is a regrowth
ratchet, not a system invariant, and D-14 says explicitly not to fabricate a binding on a test that
counts lines. Scope is `INV-PLUGIN` because both forbidden directions are the `internal/plugin`
package boundary — plugin must not import up into `grpc`, and `eventbus` must not depend on plugin.

**No in-repo analog exists for the size half.** All 17 files in `test/meta/` assert structure —
imports, proto members, config keys, invariant bindings — and none counts lines. That half is
genuinely greenfield; write it to the `test/meta` package conventions (SPDX, `package meta`,
reuse `findRepoRoot`, testify `require`) rather than citing a precedent that does not exist.

**Helper reuse is mandatory, not stylistic.** `modulePath` (`world_import_graph_test.go:18`),
`worldPkgImports` (`:24`), and `findRepoRoot` (`meta_helpers_test.go:33`) are already declared in
`package meta`. A new file in that package redeclaring any of them is a compile error. Note that
`08-RESEARCH.md`'s proposed snippet calls a `pkgProductionImports` helper that **does not exist** —
a verbatim copy will not compile. Reuse `worldPkgImports`.
