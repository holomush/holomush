<objective>
Close the two net-new live-delivery substrate gaps (Landmine 2) so `core-channels` — the FIRST consumer of session-stream subscription — can deliver live messages:

1. **Session-establishment path:** add the binary-SDK `SessionStreamsHandler` hook so a binary plugin's `QuerySessionStreams` RPC routes to plugin code (today `pluginServerAdapter` embeds `UnimplementedPluginServiceServer` → `codes.Unimplemented`). Lua parity already exists (`on_session_subscribe`, `internal/plugin/lua/host.go`), so this CLOSES a binary/Lua gap rather than opening one.
2. **Mid-session join/leave path:** serve the `stream.subscription` host capability (`StreamSubscriptionService.AddSessionStream`/`RemoveSessionStream` — proto-declared but server-stubbed as `codes.Unimplemented`, tracked `holomush-l6std`) by wiring it to the existing host `StreamRegistry` (`internal/grpc/stream_registry.go`), and add the binary-SDK client + `*Aware` interface so a plugin can call it.

Purpose: `core-channels` needs both paths — QuerySessionStreams for connect/reconnect subscription, AddStream/RemoveStream(LIVE_ONLY) for `channel join`/`leave` during a live session.
Output: SDK `SessionStreamsHandler` + `StreamSubscription` client/Aware; served `stream.subscription` capability.

**SCOPING RISK (flagged per orchestrator directive):** This is genuine host+SDK substrate plumbing, not plugin domain code. It is BOUNDED and well-scaffolded — the proto (`api/proto/holomush/plugin/host/v1/stream.proto`), the capability token (`internal/plugin/capability_vocab.go`), the auto-generated Lua bridge (`internal/plugin/luabridge/bindings_gen.go`), and the host `StreamRegistry` implementation all already exist; only the server bodies + binary-SDK client/hook are missing. If Task 2 (serving `stream.subscription`, i.e. mid-session mutation) proves larger than budgeted (e.g. the `HostCapabilities` port needs deeper threading), the DELIVERY FLOOR is Task 1 alone: session-establishment delivery via `QuerySessionStreams` still gives every member their channel streams at connect/reconnect. Mid-session live `join` would then degrade to "subscribed on next session-stream refresh" rather than instantly — a UX degradation, NOT a dropped requirement. Do NOT silently drop mid-session delivery; if it cannot land, note the degradation explicitly in the SUMMARY and file a follow-up bead against `holomush-l6std`.
</objective>

| Boundary | Description |
|----------|-------------|

| plugin capability declaration → enforcement | Consuming a capability without declaring it must fail closed |

| T-01-08 | Information disclosure | AddSessionStream targeting a foreign session | medium | mitigate | Server forwards to the host StreamRegistry which owns session validity (SESSION_NOT_FOUND on unknown session); the stream a plugin adds is a plugin-owned channel subject; membership authorization is enforced by 01-04/01-05 before the plugin ever calls AddStream |
