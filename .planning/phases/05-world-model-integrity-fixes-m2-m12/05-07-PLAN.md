<objective>
Build the single leased relay with a CONCRETE operational model (Codex Agreed-Concern B — the lease/wakeup were asserted, not mechanized), a reference idempotent consumer + bootstrap harness, and wire it as a dedicated `OutboxRelaySubsystem` through the composition root in `cmd/holomush/core.go` — replacing the dead no-emitter leg in production `world.Service` construction. Relay alerting + an operator skip/recovery affordance land in THIS slice: a halted ordered feed is otherwise silent and, without a skip path, one poison envelope kills the feed until manual SQL (Antigravity finding 15).

Concrete mechanisms this plan assigns to tasks (round-2 hardened; round-3 corrected the lease/skip OWNERSHIP contracts — blockers #2 and #3):
- Single lease = an explicit `Lease` ABSTRACTION (round-3 blocker #2). `outbox.OutboxStore` exposes only `AcquireLease(ctx, gameID) (Lease, error)`; the postgres impl acquires a session-level per-game advisory lock on a DEDICATED connection and returns a `Lease` BOUND to that connection carrying `Generation() int64` + the relay DB methods (`NextUnpublished`/`MarkPublished`/`Prune`/`MarkSkipResolved`/`CurrentEpoch`). Because the relay can only reach storage through a Lease, its queries structurally run on the lock-holding connection — the round-2 shape (pool-shaped store methods + a promise) could not prove this, since the DB provider exposes only a general pool (`internal/world/setup/subsystem.go:20-23`). On connection loss Postgres drops the lock; Lease methods fail with a typed stale-lease error and the relay must re-acquire (new generation). **Concrete generation fence (round-4 A2):** the generation is the durable `world_feed_counter.lease_generation` column (05-05); `AcquireLease` bumps it, and `MarkPublished(ctx, eventID, generation)` re-reads and compares against it — the source-of-truth and comparison point are explicit, no hand-wave.
- At-least-once, stated plainly (round-3 blocker #2): a PG lease cannot prevent a partitioned old holder from publishing a NATS message it had already issued — fencing can reject the stale holder's later DB ack (generation-checked `MarkPublished`) but cannot un-send wire traffic. Wire correctness = `Nats-Msg-Id` = event ULID (JetStream dedup window) + the idempotent reference consumer. No "split-brain impossible" claims.
- Storage seam = the consumer-owned `OutboxStore`/`Lease` interfaces declared in `internal/world/outbox` (`store.go`), implemented by the `internal/world/postgres` outbox store, injected by setup — so `internal/world/outbox` does NOT import `internal/world/postgres` (round-2 second import cycle `outbox → postgres → outbox`).
- Wakeup = a dedicated LISTEN connection; the outbox write issues `pg_notify` inside the mutation tx; a periodic sweep interval + backoff is the fallback.
- Poison recovery = HALT + a `SkipService` (round-3 blocker #3: skip needs BOTH the leased store and a JetStream publisher, so it is a relay/admin SERVICE in `internal/world/outbox/skip.go`, not a store method): acquire the fenced lease → validate the halted row → persist-or-reuse a STABLE `skip_marker_event_id` (round-4 A1 — so a crash-then-retry republishes the SAME Nats-Msg-Id instead of a fresh ULID, never two events at one feed position) → publish an operator-authorized same-feed-position MARKER (preserving gap-free wire order, INV-WORLD-3) → after PubAck mark the poison row resolved via the lease → resume. Invoked by a `holomush outbox skip` admin CLI subcommand constructed with DB + EventBus config — NOT a silent `published_at` bump, NOT a DB-only CLI (round-2: the round-1 skip left a wire gap and had no caller).
- Composition = a new `OutboxRelaySubsystem` (own `SubsystemID`, `DependsOn(Database, EventBus)`) constructed in `core.go:403`-area and registered in `productionSubsystems` (`core.go:1455`); `WorldSubsystem` gains no event-bus dependency (the relay is separate). The `productionSubsystems` count-assertion test (`core_subsystems_test.go` `allStubs [15]`→[16]) is updated.
- Import-graph guard = a meta-test asserting the FULL adjacency matrix (round-3 MEDIUM): forbidden edges world→outbox, world→postgres, outbox→postgres, outbox→world, postgres→outbox, wmodel→world, wmodel→postgres, wmodel→outbox — production imports only — PLUS the `internal/world/postgres` composition allowlist (only cmd/holomush, internal/world/setup, internal/bootstrap/setup, internal/access/setup, internal/testsupport/**, and the package itself may import it), constraining concrete world-writer construction to composition/test packages (round-3 blocker #5 pairing).
