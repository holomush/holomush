<objective>
Build the single leased relay with a CONCRETE operational model (Codex Agreed-Concern B — the lease/wakeup were asserted, not mechanized), a reference idempotent consumer + bootstrap harness, and wire it as a dedicated `OutboxRelaySubsystem` through the composition root in `cmd/holomush/core.go` — replacing the dead no-emitter leg in production `world.Service` construction. Relay alerting + an operator skip/recovery affordance land in THIS slice: a halted ordered feed is otherwise silent and, without a skip path, one poison envelope kills the feed until manual SQL (Antigravity finding 15).

Concrete mechanisms this plan assigns to tasks (round-2 hardened — several were asserted but not mechanized/were unsafe as drawn):
- Single lease = a session-level PostgreSQL advisory lock keyed by game, held on a DEDICATED connection; on connection loss Postgres drops it, and the relay re-acquires before publishing (fencing). The relay's fetch/mark/prune/skip DB queries run on THAT lease connection (not the general `r.pool`), and the publish/ack path checks a fencing generation — closing the round-2 split-brain hole (a session advisory lock alone does not fence a NATS publish issued after PG released the session lock but before the process observes the loss).
- Storage seam = a consumer-owned `OutboxStore` interface declared in `internal/world/outbox` (`internal/world/outbox/store.go`), implemented by the `internal/world/postgres` outbox store, injected by setup — so `internal/world/outbox` does NOT import `internal/world/postgres` (round-2 second import cycle `outbox → postgres → outbox`).
- Wakeup = a dedicated LISTEN connection; the outbox write issues `pg_notify` inside the mutation tx; a periodic sweep interval + backoff is the fallback.
- Poison recovery = HALT + a same-feed-position operator-authorized skip MARKER published BEFORE marking the row published (preserving gap-free wire order, INV-WORLD-3), invoked by a `holomush outbox skip` admin CLI subcommand — NOT a silent `published_at` bump (round-2: the round-1 skip left a wire gap and had no caller).
- Composition = a new `OutboxRelaySubsystem` (own `SubsystemID`, `DependsOn(Database, EventBus)`) constructed in `core.go:403`-area and registered in `productionSubsystems` (`core.go:1455`); `WorldSubsystem` gains no event-bus dependency (the relay is separate). The `productionSubsystems` count-assertion test (`core_subsystems_test.go` `allStubs [15]`→[16]) is updated.
- Import-graph guard = a meta-test asserting the cycle-free boundaries (world ↛ outbox/postgres; outbox ↛ postgres; postgres ↛ outbox).

Purpose: The outbox rows (05-05) are inert without a publisher. The relay is the only writer to JetStream for world changes; it must preserve gap-free order, dedup by `Nats-Msg-Id`, survive broker blips and reconnects without split-brain, and — unlike the audit projection it mirrors — HALT rather than skip on poison, alerting (with the halt POSITION) so operators see a stalled feed and can clear a poison row (via a same-position skip marker) without raw SQL.
Output: relay.go + the outbox-declared OutboxStore interface + the postgres store's relay-read/mark/prune/skip methods (on the lease connection) + writer NOTIFY + fencing generation, consumer.go, non-quarantined seam test, the OutboxRelaySubsystem + SubsystemID + core.go wiring + the test cascade, alerting + same-position skip marker + the admin CLI skip subcommand, the import-graph guard meta-test. Includes an automated D-04 gate-confirmation check (Task 3): a grep proving the relay imports no crypto/abac-gated file.
