<objective>
Introduce the compile-time write-requires-envelope seam — round-2 redesigned to `mutate(ctx, entity, expectedVersion, intent wmodel.EnvelopeIntent) (*wmodel.MutationDelta, error)` (intent in, delta out, envelope finalized post-write) — declare the `world.OutboxWriter` interface the executor persists through, wire `MoveCharacter` end-to-end through the outbox, and delete the post-commit emit path outright — folding WR-01 (D-03). This is the pivotal slice-2 plan: the emit-path deletion, the first outbox mover, and the resilience-test rewrite MUST land together (Pitfall 3 — deleting `events.go` breaks the integration harness that `task test` does not compile).

The round-2 review found the round-1 seam (`mutate(..., outbox.Envelope)` in package `world`) both created a Go import cycle (`world → outbox → world`) and was temporally impossible (the caller cannot build a finished envelope before the write produces the delta the manifest needs). The round-3 review then caught a residual ownership contradiction: the round-2 draft still had the EXECUTOR finalize the envelope before calling the writer that allocates the storage-owned epoch/feed_position (round-3 blocker #1). This plan adopts the fully-corrected contract: the seam names only `wmodel` value types + a `world`-owned `OutboxWriter` interface (concrete impl in `internal/world/postgres`, injected by setup), so package `world` imports neither `outbox` nor `postgres`; the executor passes `(intent, delta)` to `OutboxWriter.WriteIntent(ctx, intent, delta) (*wmodel.Envelope, error)`, and the WRITER allocates epoch/position, finalizes, persists, and returns the finalized envelope. Each envelope field has exactly one owner: intent fields ← caller; delta ← the repo write; epoch/feed_position ← the writer's counter allocation.

Purpose: The one-pager makes the guarded write seam require the envelope so that "a stated principle silently unrealized" (the drift that caused the ADR) is structurally non-recurrable. `MoveCharacter` is the empirically characterized M2 window — moving it into the same-tx outbox closes that window and proves the mechanism on a real mover before the mechanical rollout.
Output: the mutate() seam (intent in / delta out / finalization owned by the writer) + the world.OutboxWriter interface (WriteIntent), MoveCharacter through the outbox, events.go/event_store_adapter.go deleted, chaos helpers + m2 spec rewritten, f1 doc corrected.
</objective>
