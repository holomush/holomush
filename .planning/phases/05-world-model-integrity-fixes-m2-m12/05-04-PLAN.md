<objective>
Thread the read version through the read-modify-write callers (the M12 corruption path) and flip the M12 resilience spec to assert a surfaced conflict. The RMW sites — `internal/property/entity_mutator.go` `SetName`/`SetDescription` (which funnel into `UpdateLocation`/`UpdateObject`) and the inline `UpdateCharacterDescription` in `world.Service` — read an entity, mutate one field, and write the whole row back. Without carrying the read version into the guarded write, the guard never trips. This plan closes the loop and proves it via the standing D-05 resilience gate.

Purpose: The guarded repos (05-02/05-03) only reject a stale write if the caller passes the version it read. The M12 verdict's corruption path is exactly a describe/rename RMW; threading the version there is what converts "both writers return nil, one silently lost" into "the second writer gets WORLD_CONCURRENT_EDIT."
Output: version-threaded RMW callers, WORLD_CONCURRENT_EDIT surfaced at the service boundary, and the flipped M12 spec (+ a per-aggregate race spec).
</objective>
