<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| concurrent replicas → same character/object | Racing move/update writers must be serialized by the guard. |
| repo write → transaction | Object Move must not escape the ambient tx. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-07 | Tampering | character/object CAS | high | mitigate | Stale move/update affects zero rows (version mismatch); the follow-up read surfaces WORLD_CONCURRENT_EDIT. A crafted `Version` cannot bypass CAS — it must equal the stored value a genuine read returned. |
| T-05-08 | Tampering | object Move enrollment | high | mitigate | Object Move routes through `execerFromCtx` so it commits atomically with the (slice-2) outbox row; a `r.pool.Exec` Move would split the transaction. Grep gate + acceptance criteria forbid it. |
| T-05-09 | Repudiation | zero-row classifier | medium | mitigate | Locked follow-up read in the same tx makes conflict/delete/not-found deterministic. |
</threat_model>
