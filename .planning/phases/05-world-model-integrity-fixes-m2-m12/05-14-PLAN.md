<objective>
The transaction & repository foundation the reviews identified as an unaddressed prerequisite (Codex Agreed-Concern A). Three source-grounded blockers make the drawn slice-1/slice-2 mechanism unimplementable as written, and this plan fixes them once, behavior-preservingly, before the version-guard repo rewrites (05-02/05-03) and the outbox (05-05+):

1. `Transactor.InTransaction` always calls `pool.Begin` (`transactor.go:28`) — it is NOT re-entrant, so a repo method that opens a tx while its caller already holds one nests/escapes; and the zero-row classifier's "locked follow-up read in the same tx" cannot reuse the caller's connection.
2. `ExitRepository.Create`/`Delete` (`exit_repo.go:60,178`) and `ObjectRepository.Move` (`object_repo.go:172`) open their OWN `pool.Begin` transactions — they escape any ambient mutation transaction, so the outbox row would commit separately (refactored onto the re-entrant seam here). (`SceneRepository.AddParticipant`/`RemoveParticipant` also used `r.pool.Exec` directly, but round-5 D-07 REMOVES that vestigial write surface entirely rather than enrolling it — no production caller exists.)
3. Delta-parity is unbuildable from command inputs because cascade IDs (the reverse-exit ID at `exit_repo.go:80`, cascaded children) are generated INSIDE the repository; service methods only see `error`. There is no `MutationDelta` contract for repos to return.

The **round-2 review** adds four grounded requirements this plan now also lands:
4. The `MutationDelta`/`AffectedAggregate`/`EnvelopeIntent`/`Envelope` value types must live in a **cycle-neutral leaf package** (`internal/world/wmodel`) so the round-1 `world → outbox → world` and `outbox → postgres → outbox` import cycles cannot form. This plan creates `wmodel` and puts `MutationDelta`/`AffectedAggregate` there; 05-05 adds `EnvelopeIntent`/`Envelope`.
5. `ObjectRepository.Move` (`repository.go:118`) and `CharacterRepository.UpdateLocation` (`repository.go:161`) take no version-bearing struct, so the round-1 redesign gave the move/containment path no CAS parameter — add `expectedVersion int` to both.
