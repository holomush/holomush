<objective>
Persist scene notification preferences in the plugin-owned Postgres schema: a
per-character global notify on/off preference and a per-scene mute flag, with a
`mode` column that ships as the D-05 digest seam (default `realtime`). This is
the state layer that Plan 03 (telnet mute command + plugin RPCs), Plan 04 (core
mute-suppression), and Plan 05 (web mute slice) all read/write.

Purpose: mute/notify controls need durable, plugin-owned state (D-04) with a
future-proof digest seam (D-05) so digest delivery needs no later migration.
Output: migration `000011_scene_notify_prefs` (up/down) and store CRUD methods
with round-trip tests.
</objective>
