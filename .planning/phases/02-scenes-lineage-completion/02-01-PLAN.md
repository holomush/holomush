<objective>
Render scene-activity notifications on the telnet surface. The core already
emits `ControlFrame{CONTROL_SIGNAL_SCENE_ACTIVITY, scene_id}` to every
non-focused member connection (server.go:1288-1315) and the web gateway renders
it (handler.go:610-618); the telnet gateway's control switch silently drops it.
This plan adds the missing render + a per-scene throttle, backed by a NEW shared
`[>GAME: …]` Go primitive reusable by all future game-originated notices (D-03).

Purpose: makes SCENEFWD-02 true on telnet, not just web — a non-focused telnet
member is notified their scene has activity, throttled so a busy scene does not
spam one line per pose.
Output: `internal/telnet/gamenotice` primitive, a `SCENE_ACTIVITY` case in the
telnet main control loop with per-scene debounce, and INV-SCENE-70 registered.
</objective>
