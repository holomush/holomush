<objective>
Make `scene mute` AND the per-character global notify preference actually suppress
notifications. The badge downgrade at server.go:1296 is the single chokepoint where
a non-focused member's scene event becomes a content-free SCENE_ACTIVITY frame
(feeding both the web badge and the Plan 01 telnet nudge). This plan consults the
character's notify state there and drops the frame when the character's GLOBAL
notifications are OFF (review Finding 2) OR the specific scene is muted — uniformly
across both surfaces. Because CoreServer holds only the session store (no scene
handle), the state is reached through a narrow INJECTED `SceneMuteChecker` interface
(dependency inversion), TTL-cached per character so the delivery loop makes no
per-event RPC. The backing loader dispatches to the plugin with the same host-vouched
identity (actor + ownerPlayerID) the web facade uses (review Finding 3).

Purpose: without this, `scene mute` AND `SetSceneNotifyPref` (Plan 03) persist
preferences nobody reads — a broken feature (Plan 02/03/05 persist a global pref
with no consumer). This is the read/enforcement half of D-04's notification controls.
Output: a `SceneMuteChecker` interface (global-pref + mute suppression) + per-character
TTL cache, a suppression branch at the downgrade, and the concrete checker wired at
CoreServer construction with the host-vouched dispatch identity.
</objective>
