<objective>
Make `scene mute` actually suppress notifications. The badge downgrade at
server.go:1296 is the single chokepoint where a non-focused member's scene event
becomes a content-free SCENE_ACTIVITY frame (feeding both the web badge and the
Plan 01 telnet nudge). This plan consults the character's mute state there and
drops the frame for muted scenes — uniformly across both surfaces. Because
CoreServer holds only the session store (no scene handle), the mute state is
reached through a narrow INJECTED `SceneMuteChecker` interface (dependency
inversion), TTL-cached per session so the delivery loop makes no per-event RPC.

Purpose: without this, `scene mute` (Plan 03) persists a preference nobody reads —
a broken feature. This is the read/enforcement half of D-04's mute control.
Output: a `SceneMuteChecker` interface + per-session TTL cache, a suppression
branch at the downgrade, and the concrete checker wired at CoreServer construction.
</objective>
