<objective>
Give a scene participant the controls to mute a scene's notifications: telnet
`scene mute #X` / `scene unmute #X` subcommands (new — they do not exist today)
and the plugin SceneService RPCs (MuteScene, SetSceneNotifyPref, GetSceneNotifyPref,
ListMutedScenes) that the web slice (Plan 05) and the core suppression checker
(Plan 04) consume — GetSceneNotifyPref gives Plan 04 a way to READ the persisted
per-character GLOBAL notify preference so it is actually enforced (review Finding 2).
The scene-scoped mute/unmute controls are authorized by a new Layer-2
participant-gated ABAC policy, following the `end`/`pause`/`resume`
engine-Evaluate precedent; the character-self notify-pref reads/writes are
authorized by the request `character_id` ↔ host-vouched actor-metadata guard
(no `scene:<id>` — review Concern 1/2).

Purpose: D-04's notification controls — persisted, participant-gated, on both the
telnet command path (this plan) and the web typed-RPC path (Plan 05, which reuses
these same plugin RPCs via the facade).
Output: plugin scene.proto RPCs + handlers, `scene mute`/`unmute` command cases,
and the mute/unmute DSL policy.
</objective>
