<objective>
Wire the dormant idle-timeout lifecycle. The `idle_timeout_secs` column and the
`scene_idle_nudge` event type already exist but nothing drives them. This plan
adds a ticker sweep (mirroring publish_scheduler.go) that transitions idle scenes
`active → paused` and, behind an OFF-by-default flag, emits `scene_idle_nudge`.
When enabled, the nudge renders on telnet through the shared gamenotice.Idle
primitive: the telnet gateway maps the core-scenes:scene_idle_nudge EventFrame to
`[>GAME: Scene #<id> is now idle]` (review Finding 4 — the generic SCENE_ACTIVITY
downgrade renders "has new activity", NOT "is now idle", so the idle nudge needs
its own explicit render path). Non-focused members still receive the content-free
SCENE_ACTIVITY downgrade (Plan 01's generic leader) by INV-SCENE-62 privacy design.

Purpose: D-06 — idle-timeout defaults + transitions + optional idle nudge, using
the existing partial infra, not a rebuild.
Output: idle_scheduler.go sweep + store query (default supplied as an explicit
parameter — review Finding 1) + game-default config + the emitter call + the telnet
gamenotice.Idle render branch (review Finding 4), and INV-SCENE-71 for the idle transition.
</objective>
