<objective>
Extract the gRPC **client** out of `internal/grpc` into a new dependency-free
package `internal/grpcclient`, so the gateway can hold a gRPC client without
dragging the CoreServer monolith's 41-package closure into `internal/telnet`.

Purpose: this is ~85% of ARCH-05's real value and it is the **only** violation
whose closure actually reaches the DB. `internal/telnet` today has a 47-package
internal closure (`go list -deps ./internal/telnet` → 47) purely because
`gateway_handler.go:29` imports `internal/grpc` for one function.

Output: `internal/grpcclient` (a true leaf — proto + grpc-go + oops only), with
`internal/grpc/client.go` deleted and every caller rewired.

Settles: D-16's `TranslateSubscribeErr` row (destination is Claude's discretion
per CONTEXT.md), plus a violation D-16's inventory **omits** — see
`<drift_correction>`.
</objective>

<drift_correction>
**CONTEXT.md D-16 and RESEARCH.md Pitfall-4 are both wrong about the blast radius.**

RESEARCH.md Pitfall 4 asserts: *"Already verified safe. I enumerated all 38
`cmd/holomush` files not in `coreOnlyFiles` and checked each for
`internal/{core,session,grpc}` imports: **zero hits**."*

That is false. Verified live in this worktree:

- `cmd/holomush/gateway.go:20` imports `holoGRPC "github.com/holomush/holomush/internal/grpc"`.
- `gateway.go` is **not** in `coreOnlyFiles` (`cmd/holomush/gateway_imports_test.go:22-99`).
- It uses `holoGRPC.ClientConfig` (`:152`, `:227`) and `holoGRPC.NewClient` (`:153`).

So D-17's "just add `internal/grpc` to `forbidden`" **fails** unless either
(a) `gateway.go` is added to `coreOnlyFiles` — the exact escape hatch D-15
explicitly rejects, and a lie (`gateway.go` **is** the gateway), or
(b) the client is extracted. This plan does (b).

Five gateway **test** files also import `internal/grpc` and are in scope because
`gateway_imports_test.go:123` sets `Tests: true`:
`internal/web/handler_test.go:25`, `internal/web/scene_handlers_test.go:19`,
`internal/telnet/gateway_handler_test.go:27` (all three for
`var _ CoreClient = (*holoGRPC.Client)(nil)` compile-time assertions).
The extraction fixes these too.

Do NOT restate CONTEXT.md D-16's claim that *"`internal/grpc` is the CoreServer
monolith … so a gateway file drags the whole domain into its closure while
passing the tripwire."* The closure half is true and verified (47 packages); the
"while passing the tripwire" half is what this phase fixes. State the closure
fact, not the editorial.
</drift_correction>
