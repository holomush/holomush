<objective>
Collapse the two system-broadcast implementations into **one builder with two
callers** (D-02), so the `{"message": ...}` payload shape exists in exactly one
place — and so `internal/command` can shed its event dependency entirely rather
than importing `internal/eventbus` (D-01).

Purpose: `internal/plugin/hostcap/system_broadcaster.go:45-48`'s doc comment
admits it *"Mirrors the `{"message": ...}` payload shape of
`command.Services.BroadcastSystemMessage`"* — a drift contract enforced by
nothing. Verified live: `internal/command/types.go:622-641` builds the identical
payload independently.

Output: `internal/sysbroadcast` (the one builder), a one-method port in
`internal/command`, and `Services.Events()` deleted.
</objective>

<constraints>
**`internal/command` MUST NOT import `internal/eventbus`** (D-01) — importing it
would deepen the exact plugin⇄eventbus⇄grpc coupling arch-review MEDIUM-4 wants
unwound. Verified live today: `go list -deps ./internal/command` contains
`internal/eventbus/audit/auditheader` (a leaf) but **not** `internal/eventbus`.
That must still be true after this plan. `internal/command` reaches the
broadcaster through a **consumer-defined interface it declares itself** — the
Go idiom, and D-02's explicit instruction ("Consumer-defined interfaces — no
shared port package").

**FINDING-5 applies here too (BLOCKING, cross-AI review): a `Publisher` alone cannot
qualify a subject.** Both external reviewers found this independently on 07-05 AND
07-06. `eventbus.Qualify` rejects a relative ref with an empty game id
(`qualify.go:29-31`) and `NewSubject` rejects any subject not starting with `events.`
(`types.go:257-258`) — so a `Broadcaster` holding only an `eventbus.Publisher` would
fail at publish on the reserved `"system"` subject, every time. `Broadcaster`
therefore carries `gameID func() string` exactly as `presence.Emitter` does (07-05
§ FINDING-5), including the `"" → "main"` fallback. Read 07-05's settled shape and
match it — the two builders MUST qualify identically, and 07-06 depends on 07-05 so
`internal/presence/emitter.go` exists to copy from.

⚠️ **Do NOT "fix" the Lua/binary asymmetry while re-typing the port.**
`internal/plugin/setup/subsystem.go:490-492`'s doc states: *"The binary host
needs no equivalent: SessionAdminService is Lua-only (not in
`hostcap.BinaryDefaultSet`)."* This is a **pre-existing, documented, permitted
asymmetry** per `.claude/rules/plugin-runtime-symmetry.md` § "Permitted
asymmetry: same capability, different transport". Re-typing the sink MUST NOT
change which runtimes reach `SessionAdmin`. Adding a binary SessionAdmin surface
is out of scope and would be a new capability, not a refactor.
</constraints>
