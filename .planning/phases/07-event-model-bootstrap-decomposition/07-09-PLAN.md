`topoSort` + `DependsOn` (`internal/lifecycle/orchestrator.go:44-149`) **already

MEDIUM-11 failure mode (comments + slice order asserting an order the graph did
not enforce; the graph silently wins). If a readiness gate is ever wanted, reuse
`lifecycle.ReadinessRegistry` (`internal/lifecycle/registry.go` —
`AllReady`/`WaitReady`/`HealthReporter`, already used at core.go step 9). Do not
invent one.
