<objective>
Fix a **live correctness bug** (D-07): plugin history pagination is ULID-ordered
in a system whose own code states ULID order does not match stream order. On a
busy stream, plugin history pages can **skip or repeat** events.

Purpose: D-07 is not tidiness. Verified live, hop by hop:
1. `QueryHistory` returns `eventbus.Event` **with** `Seq`
   (`internal/eventbus/types.go:141-148`).
2. `busEventToCoreEvent` **destroyed** it — `core.Event` had no such field
   (deleted in 07-07, which is what unblocks this plan).
3. `encodeHostEventCursor` is therefore forced to hardcode `Seq: 0`, and
   **says so** (`internal/plugin/hostcap/servers.go:1279-1296`).
4. `busHistoryReaderAdapter.ReplayTail` (`cmd/holomush/sub_grpc.go:916-935`)
   passes only `Subject`/`Direction`/`PageSize`/`NotBefore`/`BeforeID` — no seq.
5. `internal/eventbus/history/hot_jetstream.go:424-431`: *"Ordering is owned by
   JetStream per-stream sequence, not ULID lex order — concurrent publishers
   produce events whose ULIDs do NOT match stream sequence."*

**This IS a behavior change** (pagination becomes seq-correct) and MUST ship with
a regression test pinning the concurrent-publisher skip/repeat case.

Output: a genuine RED→GREEN seq fix, plus a guard that keeps `Seq` off the
plugin wire (D-08).
</objective>

### D-07's signature shape (Claude's discretion per CONTEXT.md): a `beforeSeq uint64` param

`ReplayTail` becomes:
`ReplayTail(ctx context.Context, stream string, count int, notBefore time.Time, beforeSeq uint64, beforeID ulid.ULID) ([]eventbus.Event, error)`

Chosen over a cursor struct because the existing parameter list already carries
the cursor pieces positionally (`notBefore`, `beforeID`), a struct would churn
all 10 sites (2 interface decls + 4 impls + 4 test fakes) for no contract gain,
and `beforeSeq` sits naturally beside `beforeID` — the pair that travels
together in `cursor.HostCursor{Seq, ID}`.

<do_not_reintroduce>
⚠️ **D-08 — do NOT expose `Seq` to plugins.** `hostv1.Event`
(`api/proto/holomush/plugin/host/v1/stream.proto:45-71`) has **8 fields and no
seq** — verified live. The `cursor` field (`= 8`) is documented there as an
**opaque** blob, encoded by `internal/eventbus/cursor`, an `internal/` package an
external plugin cannot import. Plugins therefore see **no seq today**, hardcoded
or otherwise, so D-07 is a **zero-contract-change** fix.

Adding a plugin-facing `seq` would (a) violate `internal/eventbus/types.go:143`
(*"Seq … Host-internal — never serialized in any public proto envelope"*), and
(b) hand plugins a number whose meaning silently changes across tiers —
`history/tier.go:713-718`: a cold event's `js_seq` is an aged-out JS seq no
longer in JS retention; seq is only meaningful within a tier's sequence space.
A future plugin needing independent ordering requires an explicit invariant
amendment + ADR, not a silent override.
</do_not_reintroduce>
