### MEDIUM-11 — SETTLED as a **real edge**, because the intent is real and security-bearing

CONTEXT.md leaves this to the planner ("a real `DependsOn` edge **or**
comment-deletion + a topo-order pin test — decide against the actual boot-order
intent"), and RESEARCH notes the intent must be established first because "the
comment asserts an order the graph contradicts, so one of them is wrong".

**The comment is wrong about the graph; the intent behind it is right.** Verified
live:
- `cmd/holomush/core.go:1476` — `cryptoChainVerifierSub, // runs before EventBus (chain integrity check)`.
- `internal/eventbus/subsystem.go:77` — `func (s *Subsystem) DependsOn() []lifecycle.SubsystemID { return nil }` → in-degree 0 → `topoSort` **seeds it first** (`orchestrator.go:118-125`).

⇒ EventBus starts **before** the verifier — the opposite of the comment.

The intent is grounded, not editorial: the verifier enforces **INV-CRYPTO-102**
(`verifier_subsystem.go:74-77`) — *"server boot MUST refuse with
AUDIT_CHAIN_BROKEN (or a more specific AUDIT_CHAIN_* code) when any registered

while the existing audit chain is unproven.

**Add the real edge:** `eventbus.Subsystem.DependsOn()` returns
`[lifecycle.SubsystemCryptoChainVerifier]`. No cycle (Verifier→Database,
EventBus→Verifier, Database→∅). **Verified safe:** `lifecycle.NewOrchestrator`
has exactly one production registration site — `cmd/holomush/core.go:1086-1098`,
which always registers both — plus `internal/lifecycle/orchestrator_test.go`'s
stubs. `internal/testsupport/integrationtest/` does **not** use the lifecycle
orchestrator (it hand-wires), so no harness can trip `topoSort`'s
"missing dependency" error by registering EventBus without the Verifier.

Delete the comment's ordering claim from `core.go:1476` — slice position asserts
nothing once `topoSort` re-derives the order, and leaving prose that duplicates
an edge is how this drifted in the first place. Replace it with a pointer to the
edge.

### LOW-7 — SETTLED: `StopAll` takes the deadline AND honors it

`cmd/holomush/core.go:1105` is `defer orch.StopAll(context.Background())` — **no
deadline**. A hung plugin subprocess or NATS drain means the process never
self-exits (mitigated only externally by systemd/k8s). `StopAll`
(`orchestrator.go:81-96`) returns nothing and never checks `ctx.Done()`.

Pass the existing 5s `shutdownCtx` **and** make `StopAll` honor `ctx.Done()`.
Passing the ctx alone is insufficient: `Stop` implementations that ignore it
would still hang the loop.

⚠️ `StopAll` is also called by `StartAll`'s rollback path
(`orchestrator.go:60`), so any signature change ripples there. Keep
`StopAll(ctx)`'s signature; change only its body.
</settlements>
