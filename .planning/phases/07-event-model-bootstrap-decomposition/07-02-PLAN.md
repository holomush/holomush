<objective>
Create `internal/eventvocab` — the dependency-free event-type vocabulary leaf
(D-05) — and move the event-type constants and event payload structs out of
`internal/core` into it.

Purpose: D-05 resolves a collision **neither ARCH-04 nor ARCH-05 mentions**. The
telnet gateway renders on `core.EventTypeArrive`/`core.EventTypeLeave`
(`internal/telnet/gateway_handler.go:1247-1249`, verified live). If the
vocabulary followed the Event into `internal/eventbus`, telnet would have to
import `internal/eventbus` — already on the `forbidden` list
(`cmd/holomush/gateway_imports_test.go:106`). Fixing ARCH-04 naïvely would make
ARCH-05 strictly worse. Only the vocabulary goes low; the struct still collapses
into `eventbus.Event` (07-07).

Output: `internal/eventvocab` (leaf), with `internal/core` and all 34 consumers
repointed.
</objective>

<scope_notes>
**`core.CommandResponsePayload` belongs here, not in `internal/presence`.**
It lives at `internal/core/engine.go:17-19` purely by file accident — it is the
payload for the `command_response`/`command_error` event types, consumed by
`internal/grpc`'s `emitCommandResponse` and by
`internal/telnet/gateway_handler_test.go:371`. It is **not** Engine's. A naïve
"move engine.go → presence" would drag an `internal/grpc` payload type into
`internal/presence`. It is vocabulary; it goes in this leaf.

**`ArrivePayload` / `LeavePayload` do NOT come here** — D-04 states they travel
with the Emitter into `internal/presence` (07-05).

**`core.Actor` / `core.ActorKind` / `core.ActorSystemID` / `core.SystemActorULID`
do NOT come here.** Verified live: `internal/command/dispatcher.go:427` calls
`core.WithActor(ctx, core.Actor{...})` and `internal/command` MUST NOT import
`internal/eventbus` (D-01). `core.Actor` is the actor-**context** vocabulary, not
Event vocabulary, and D-01's deletion list names only `core.Event`,
`core.NewEvent`, and `core.EventAppender` — not `core.Actor`. It stays in
`internal/core`. No gateway file references it (verified: the complete gateway
`core.*` census is `EventTypeArrive`, `EventTypeLeave`, `NewULID`,
`ParseCommand`, `CommandResponsePayload`, `EventTypeCommandResponse`, and the
`LocationState*`/`ExitUpdate*`/`OOC`/`Page`/`Pemit` payloads).

**`core.Event` and `core.NewEvent` do NOT come here** — they are deleted in
07-07. This plan leaves them in `internal/core`, re-typed against
`eventvocab.EventType`. `internal/core` importing `internal/eventvocab` keeps
`internal/core` a leaf-of-leaves; it does not make it a domain package.
</scope_notes>
