<objective>
Settle the external-mode configuration vocabulary and add fail-closed config
validation. Reconcile the pre-existing (unimplemented) `ModeCluster`/`cluster_url`/
`credentials_file` names to D-01's `external`/`url`/`credentials`, add the
TLS/creds/provision/DLQ config surface, and introduce `Config.Validate()` so a
`mode: external` config with no URL fails at config-validation time rather than
at dial time.

06 (boot self-check) all consume the config keys settled here. Getting the naming
reconciliation right first prevents downstream drift (RESEARCH A5).
Output: `internal/eventbus/config.go` with `ModeExternal`, reconciled
`URL`/`Credentials`, nested `TLSConfig`/`DLQConfig`, `Provision`, and a
`Validate()` method wired into core boot.
</objective>
