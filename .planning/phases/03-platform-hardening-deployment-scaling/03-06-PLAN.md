<objective>
Ship the single-principal subject-scoping story for external mode: (a) `deploy/nats/`
account templates granting `holomush-server` exactly the game-topic prefixes, (b) an
external verification script proving a non-server principal is denied, and (c) a
boot-time self-check that refuses to start if the server's own account is over-scoped.
All three, complementary (D-13).

Purpose: CLUSTER-02. The scoping design (phase3d-grounding) enforces single-principal
publish/subscribe at the NATS account layer; single-principal can only be fully proven
from BOTH sides — the server self-verifies it is not over-scoped, and the external
script verifies other principals are locked out.
Output: the `deploy/nats/` assets + `scopecheck.go` + integration proof + core wiring.
</objective>
