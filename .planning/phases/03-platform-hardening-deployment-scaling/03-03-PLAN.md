<objective>
Implement the `mode: external` branch inside `eventbus.Subsystem.Start`: dial an
external NATS URL with `.creds`/TLS instead of embedding a server, fail closed at
boot if unreachable, guard the embedded-only Prometheus exporter, and add the
`provision: false` verify-or-fail seam for locked-down clusters.

Purpose: CLUSTER-01's core code delta. The cluster/invalidation/audit stack is
transport-agnostic (core-NATS via `natsconn.Conn`), so switching to external NATS
is almost entirely a connection-establishment change — split the mode-dependent
connect from the mode-independent stream declaration and everything downstream
keeps working (RESEARCH OQ-1).
Output: `natsdial.go` + a refactored `Start` with a `connect()` mode switch and a
provision opt-out branch, plus an integration proof against a real NATS container.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| core process ↔ external NATS | Network hop; authn via `.creds`/TLS; the server's account is the sole game-topic principal |
| operator config → provisioning authority | provision flag controls whether the server may run `$JS.API` stream admin |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-05 | Spoofing / Info-disclosure | external NATS connection without authn/encryption | high | mitigate | `.creds` (JWT/NKey) auth via `nats.UserCredentials`; optional mTLS via the TLS block (nats.go RootCAs/ClientCert) — do NOT hand-roll (D-04, ASVS V2/V6) |
| T-03-06 | Denial of Service / policy bypass | external NATS unreachable → insecure degraded fallback | high | mitigate | Fail closed at boot (`EVENTBUS_EXTERNAL_CONNECT_FAILED`); no embedded fallback; orchestrator owns retry (D-02) |
| T-03-07 | Elevation of Privilege | server given `$JS.API` stream-admin it should not hold | medium | mitigate | provision:false verify-or-fail seam lets operators deny stream admin to the server account and still boot (D-03) |
| T-03-08 | Tampering | stream config drift on a locked-down cluster silently accepted | medium | mitigate | provision:false compares StreamInfo.Config and fails closed on mismatch (`EVENTBUS_STREAM_CONFIG_MISMATCH`) |
</threat_model>
